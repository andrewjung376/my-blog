## Claude Code란?

Claude Code는 Anthropic이 공식 제공하는 AI 코딩 CLI 도구입니다. 터미널에서 직접 Claude AI와 대화하며 코드 작성, 디버깅, 리팩토링, 문서화 등 소프트웨어 개발 작업을 수행할 수 있습니다.

### 주요 특징
- 터미널에서 직접 실행 (CLI)
- 파일 읽기/쓰기/편집 가능
- Git 작업 지원
- MCP(Model Context Protocol) 서버 연동
- IDE 확장 프로그램 지원 (VS Code, JetBrains)
- 웹 앱 (claude.ai/code) 지원

---

## 설치 및 시작

### 설치
```bash
npm install -g @anthropic-ai/claude-code
```

### 실행
```bash
# 현재 디렉토리에서 시작
claude

# 특정 파일과 함께 시작
claude <파일명>

# 단일 프롬프트 실행 후 종료
claude -p "할 작업 설명"
```

### 로그인
```bash
# API 키로 로그인
claude login
```

## 기본 사용법

Claude Code는 자연어로 지시를 내리면 됩니다.

### 예시 명령어
```
# 코드 작성 요청
"React 로그인 컴포넌트를 만들어줘"

# 버그 수정
"auth.js 파일의 JWT 토큰 만료 버그를 찾아 고쳐줘"

# 코드 설명
"이 함수가 어떻게 동작하는지 설명해줘"

# 리팩토링
"이 코드를 TypeScript로 변환하고 타입을 추가해줘"

# 테스트 작성
"이 모듈에 대한 단위 테스트를 작성해줘"
```

### 멀티턴 대화
Claude Code는 대화 맥락을 기억합니다. 이전 대화 내용을 참고하여 후속 작업을 요청할 수 있습니다.

## 슬래시 커맨드

대화 중 `/`로 시작하는 특수 명령어를 사용할 수 있습니다.

| 커맨드 | 설명 |
|--------|------|
| `/help` | 도움말 표시 |
| `/clear` | 대화 기록 초기화 |
| `/model` | 사용할 AI 모델 변경 |
| `/fast` | Fast 모드 토글 (빠른 응답) |
| `/review` | 코드 리뷰 실행 |
| `/code-review ultra` | 멀티 에이전트 심층 코드 리뷰 |
| `/code-review ultra <PR번호>` | GitHub PR 코드 리뷰 |
| `/permissions` | 권한 설정 |
| `/config` | 설정 보기/변경 |
| `/hooks` | 훅 설정 |
| `/agents` | 에이전트 설정 |
| `/doctor` | 환경 진단 |
| `/login` | 로그인 |
| `/logout` | 로그아웃 |

## 파일 및 코드 작업

Claude Code는 프로젝트 파일에 직접 접근하여 작업합니다.

### 파일 읽기/작성
- 파일 내용을 자동으로 읽고 문맥을 파악합니다
- 코드 변경 시 직접 파일을 편집합니다
- 새 파일 생성도 가능합니다

### 코드베이스 탐색
```
# 특정 기능 찾기
"프로젝트에서 인증 관련 코드를 찾아줘"

# 의존성 분석
"이 컴포넌트가 어디서 사용되는지 찾아줘"

# 아키텍처 이해
"이 프로젝트의 전체 구조를 설명해줘"
```

### CLAUDE.md 파일
프로젝트 루트에 `CLAUDE.md` 파일을 만들면 Claude Code에게 프로젝트 특정 지침을 제공할 수 있습니다.

```markdown
# CLAUDE.md 예시

## 프로젝트 개요
이 프로젝트는 Next.js 기반의 블로그입니다.

## 코딩 규칙
- TypeScript 사용
- 함수형 컴포넌트만 사용
- CSS Modules 사용

## 주요 명령어
- 개발 서버: npm run dev
- 빌드: npm run build
- 테스트: npm test
```

## MCP 서버 연동

MCP(Model Context Protocol)를 통해 외부 서비스와 연동할 수 있습니다.

### MCP란?
- Claude Code의 기능을 확장하는 표준 프로토콜
- 데이터베이스, API, 외부 서비스 등과 연결 가능

### 주요 MCP 서버 예시
- **파일 시스템**: 특정 디렉토리 접근
- **GitHub**: PR, 이슈 관리
- **데이터베이스**: SQL 쿼리 실행
- **Slack**: 메시지 전송
- **Notion**: 문서 관리

### MCP 설정 방법
Claude Code 설정 파일에 MCP 서버를 추가합니다.

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "your-token"
      }
    }
  }
}
```

## 설정 및 커스터마이징

### 설정 파일 위치
- **전역 설정**: `~/.claude/` 디렉토리
- **프로젝트 설정**: 프로젝트 루트의 `.claude/` 디렉토리

### 주요 설정 항목

| 설정 | 설명 |
|------|------|
| 모델 선택 | 사용할 Claude 모델 지정 |
| 권한 모드 | 파일 접근 및 명령 실행 권한 |
| 자동 승인 | 특정 작업의 자동 승인 여부 |
| 훅(Hooks) | 특정 이벤트에 실행할 셸 명령 |

### 훅(Hooks) 설정
훅을 사용하면 Claude Code의 특정 이벤트(파일 수정, 명령 실행 등)에 자동으로 스크립트를 실행할 수 있습니다.

## 고급 기능

### 에이전트 SDK
Claude Code를 기반으로 커스텀 AI 에이전트를 만들 수 있습니다.

### 서브에이전트 (멀티 에이전트)
복잡한 작업을 여러 에이전트로 분산하여 병렬 처리합니다.
```
"이 대규모 마이그레이션 작업을 병렬로 실행해줘"
```

### 컨텍스트 관리
긴 대화에서 컨텍스트가 압축되어 작업을 계속 이어갈 수 있습니다.

### 메모리 시스템
`~/.claude/projects/` 디렉토리에 프로젝트별 메모리를 저장하여 대화 간 컨텍스트를 유지합니다.

## 팁과 모범 사례

### 효과적인 프롬프트 작성

1. **구체적으로 설명하기**
   ```
   ❌ "코드 고쳐줘"
   ✅ "user.js 파일의 45번째 줄에서 undefined 에러가 발생해. checkAuth 함수를 수정해줘"
   ```

2. **맥락 제공하기**
   ```
   ✅ "React 18과 TypeScript를 사용하는 프로젝트야. useState 훅을 사용해서 폼 상태를 관리하는 컴포넌트를 만들어줘"
   ```

3. **단계적으로 요청하기**
   - 한 번에 너무 많은 변경을 요청하지 말고 단계적으로 진행
   - 각 단계가 완료되면 다음 단계 진행

### 권한 관리
- 필요한 최소한의 권한만 허용
- 파괴적인 작업(파일 삭제, 데이터베이스 변경 등)은 신중하게 승인

### 코드 검토
- Claude Code가 만든 코드는 반드시 검토
- 특히 보안과 관련된 코드는 더 꼼꼼히 확인

### CLAUDE.md 활용
- 프로젝트 컨벤션, 기술 스택, 주요 명령어를 CLAUDE.md에 문서화
- 팀 공유 가능
